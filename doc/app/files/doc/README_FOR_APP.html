ระบบสารสนเทศบูรณาการเพื่อการบริหารงานองค์กรปกครองส่วนท้องถิ่น (eLocal)

หลักการและเหตุผล

ระบบ eLocal เป็นการนำเอาเทคโนโลยีสารสนเทศมาใช้เพิ่มประสิทธิภาพในการปฏิบัติราชการ และให้บริการประชาชนสำหรับองค์กรปกครองส่วนท้องถิ่น ส่งผลดีในด้านการบริหารจัดการข้อมูล การจัดเก็บรายได้ และความพึงพอใจของผู้ใช้บริการ มีการบูรณาการข้อมูลเพื่อให้ผู้บริหาร และเจ้าหน้าที่ที่เกี่ยวข้อง สามารถนำข้อมูลไปใช้ในการดำเนินงานกำหนดนโยบาย ติดตามและเร่งรัดการปฏิบัติตามแผนงานโครงการต่างๆ ตลอดจนถึงการให้บริการประชาชนในลักษณะของศูนย์บริการร่วม และบริการต่างๆของเทศบาลผ่านทางออนไลน์ เพื่อเพิ่มศักยภาพในการดำเนินการให้มีประสิทธิภาพดียิ่งขึ้น

วัตถุประสงค์

  • ผู้บริหารและเจ้าหน้าที่ผู้ปฏิบัติงานสามารถเข้าถึงสารสนเทศที่ต้องการได้อย่างรวดเร็ว ทันต่อเหตุการณ์ เนื่องจากข้อมูลถูกจัดเก็บ และบริหารอย่างเป็นระบบ
  • ผู้บริหารสามารถกำหนดเป้าหมายกลยุทธ์ และวางแผนปฏิบัติการ โดยมีข้อมูลจากระบบสารสนเทศ ช่วยในการตัดสินใจ
  • มีการบันทึกข้อมูลจากการปฏิบัติงานทุกภาคส่วน ทำให้มีประวัติของข้อมูลอย่างต่อเนื่อง สามารถที่จะบ่งชี้แนวโน้มของการดำเนินงานได้อย่างถูกต้อง
  • มีระบบเพื่อติดตามผลการปฏิบัติราชการตามนโยบาย และยุทธศาสตร์ของหน่วยงานในสังกัดได้อย่างมีประสิทธิภาพ
  • ช่วยในการศึกษา วิเคราะห์สาเหตุของปัญหา และวางแนวทางแก้ไข กรณีที่การดำเนินงานไม่เป็นไปตามแผนงาน โดยการวิเคราะห์ข้อมูลที่เกิดขึ้น
  • เพื่อพัฒนาให้มีบริการประชาชนที่รวดเร็ว ถูกต้อง ในขณะที่ลดการทำงานอันซ้ำซ้อน เพิ่มความพึงพอใจให้กับประชาชนผู้ใช้บริการที่มีประสิทธิภาพมากขึ้น
  • สามารถจัดเก็บรายได้ได้อย่างถูกต้อง ครบถ้วน และลดค่าใช้จ่ายในการประมวลผลเอกสาร
  • ลดค่าใช้จ่ายในการทำงาน จากการลดเวลาการทำงานที่ซ้ำซ้อน การค้นหาข้อมูล และการจัดทำรายงานสรุปโดยอัตโนมัติ
  • มีระบบแลกเปลี่ยนข้อมูลด้วยเทคโนโลยีเว็บเซอร์วิสกับระบบงานในปัจจุบัน และเว็บท่าขององค์การบริหารส่วนตำบล… และหน่วยงานที่เกี่ยวข้องทั้งส่วนกลาง และส่วนภูมิภาค ตลอดจนภาคธุรกิจ

เป้าหมาย

  • พัฒนาระบบสารสนเทศบูรณาการเพื่อการบริหารงานองค์กรปกครองส่วนท้องถิ่น ตามขอบเขตการดำเนินโครงการ
  • จัดอบรมผู้บริหาร เจ้าหน้าที่ผู้เกี่ยวข้อง และเจ้าหน้าที่ผู้ดูแลระบบ
  • องค์การบริหารส่วนตำบล… มีระบบสารสนเทศที่ใช้ในการบริหารจัดการองค์กรอย่างมีประสิทธิภาพ

ขอบเขตการดำเนินโครงการ

  • ศึกษา วิเคราะห์ความต้องการใช้งานเทคโนโลยีสารสนเทศขององค์การบริหารส่วนตำบล… และจัดทำข้อกำหนดความต้องการของระบบ (Software Requirement Specification)
  • ออกแบบระบบสารสนเทศบูรณาการเพื่อการบริหารงานองค์การบริหารส่วนตำบล… โดยใช้แผนภาพ Mind Map ในการออกแบบข้อกำหนดรูปแบบการพัฒนา (Software Design Specification) และออกแบบหน้าจอติดต่อกับผู้ใช้ตลอดจนรายงานเอกสารทั้งหมด โดยครอบคลุมส่วนงานต่างๆอย่างน้อยดังต่อไปนี้

ส่วนบริหาร

  • ระบบสารสนเทศผู้บริหาร (Executive Information System)
  • ระบบงานการเลือกตั้ง
  • ระบบข้อมูลพื้นฐาน
  • โครงสร้างประชากร
  • การประกอบอาชีพ
  • ผลผลิตทางการเกษตร
  • ข้อมูลผู้ประกอบการ
    • ภาคการผลิต
    • ภาคบริการ

สำนักงานปลัด

  • งานนโยบายและแผน
  • งานสารบรรณ
  • ระบบจัดการประชุม
  • ระบบบริหารจัดการองค์ความรู้
  • ระบบงานบุคคลากร
    • การลา
    • ประวัติการฝึกอบรม
    • ประเมินผลการปฏิบัติงาน
    • การดำเนินการทางวินัย
  • ระบบบริหารจัดการขยะมูลฝอย และสิ่งปฏิกูล
  • ระบบงานใบอนุญาต
    • จัดตั้งสถานสะสมอาหาร
    • จัดตั้งสถานที่จำหน่ายอาหาร
    • กิจการที่เป็นอันตรายต่อสุขภาพ
  • ระบบงานฌาปนกิจ
  • ระบบงานเบี้ยยังชีพ (ผู้สูงอายุ, ผู้พิการ, และผู้ป่วยโรคเอดส์)
  • ระบบบริหารข้อร้องเรียน

ส่วนการคลัง

  • งานจัดเก็บรายได้
  • ตลาด
  • ระบบทะเบียนทรัพย์สิน
  • ระบบข้อมูลเอกสารสิทธิ์ที่ดิน สค ๑, ทบ ๕, นส ๓, นส ๓ก, นสร., และโฉนด,
  • งานบัญชีและการเงิน
  • ระบบเงินเดือนพนักงานจ้าง
  • งานบริหารพัสดุ และครุภัณฑ์
  • ระบบเบิกจ่ายฎีกา

ส่วนโยธา

  • งานควบคุมโครงการ
  • ระบบใบอนุญาตก่อสร้าง
  • งานประปา

ส่วนการศึกษา ศาสนา และวัฒนธรรม

  • ระบบการบริหารจัดการศูนย์พัฒนาเด็ก
  • ระบบข้อมูลสถานศึกษา
  • ส่วนบริการประชาชน
  • ระบบข้อมูลกลุ่มเกษตรกร
  • ระบบข้อมูลอาสาสมัครป้องกันภัยฝ่ายพลเรือน
